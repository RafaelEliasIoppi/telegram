Visão geral

O Bot Hub fica observando 3 tipos de coisa:

  1. Defesa Civil RS e INMET — fontes nativas, já vêm configuradas;
  2. Gmail — checa sua caixa atrás de assuntos cadastrados em /filtros;
  3. Fontes Customizadas — qualquer site ou RSS que você adicionar em /fontes.

Quando encontra algo novo, manda automaticamente pelo Telegram para os chats em /chats.

Painel de Controle (/dashboard)
  • Status OPERACIONAL/ATENÇÃO resume se há ao menos uma fonte ativa.
  • Stat cards: alertas de hoje, total recentes, chats ativos, webhook, última verificação.
  • Painel Fontes de Monitoramento: mostra cada fonte com seu estado e quantos alertas gerou em 7 dias.
  • Verificar Agora: força um ciclo de verificação imediato em todas as fontes — útil pra testar.
Histórico (/historico)

Lista os últimos 50 alertas com filtro por fonte/nível. Útil pra auditar e descobrir falsos positivos.

Chats (/chats)
  • Cada linha é um destinatário do Telegram (pessoa ou grupo).
  • Cadastre o Chat ID numérico do destinatário. Descubra mandando /start ao seu bot e abrindo https://api.telegram.org/bot<TOKEN>/getUpdates, ou usando @userinfobot.
  • Nível Mínimo: filtra o ruído. Se setar CRITICO, o chat só recebe alertas críticos.
  • Fontes Ativas: limita quais fontes esse chat aceita. Vazio = todas.
Filtros de Assunto Gmail (/filtros)
  • Adicione quantos trechos de assunto quiser. O monitor casa o assunto dos e-mails novos contra todos os filtros ativos.
  • Busca ignora acentos e caixa. Match parcial.
  • O botão Pausar desativa sem apagar. Bom pra testar.
  • O corpo do e-mail é parseado: rótulos Remetente:, Assunto:, frase caixa de entrada da X; quando casa, é renderizado em Markdown bonito.
Fontes Customizadas (/fontes)

Cadastre qualquer URL ou feed RSS. Para cada fonte:

  • Tipo URL: o bot baixa a página com Jsoup, varre h1/h2/h3 + parágrafos por padrão. Pode passar um seletor CSS (ex.: .noticia) pra restringir.
  • Tipo RSS: varre <item> / <entry>.
  • Palavras-chave separadas por vírgula: pelo menos uma precisa aparecer no título ou descrição. Vazio = pega tudo.
  • Nível: define a cor/prioridade do alerta no Telegram.
Bot (/bot)
  • Mostra o token configurado, o bot conectado (via getMe) e os comandos suportados.
  • Registrar Webhook: ativa modo webhook (precisa de TELEGRAM_WEBHOOK_URL apontando pra URL pública do app).
  • Remover Webhook: volta pro modo manual/polling. Pra alertas só enviados, modo polling já basta.
Configurar Telegram
  1. Crie um bot com @BotFather e copie o token.
  2. Defina o env TELEGRAM_BOT_TOKEN e TELEGRAM_CHAT_IDS (vírgula separa).
  3. Inicie uma conversa com seu bot antes — o Telegram não permite enviar mensagem a quem nunca te falou.
Configurar Gmail
  1. Ative a verificação em 2 etapas em myaccount.google.com/security.
  2. Gere uma senha de app em myaccount.google.com/apppasswords.
  3. Acesse /gmail para configurar e-mail, senha e ativar o monitor.
  4. Cadastre os assuntos a observar em /filtros.

As credenciais são salvas no banco local e podem ser alteradas a qualquer momento pela interface. Também é possível usar as variáveis de ambiente GMAIL_USER e GMAIL_APP_PASSWORD como fallback.

Quando algo não funciona
  • Nenhum alerta chega → cheque se há ao menos um chat ativo (/chats) cujo nível mínimo aceita o nível do alerta.
  • Verificar Agora não traz nada novo → normal se não houve mudança nas fontes desde o último ciclo. O dedup (SHA-256 + Message-ID) impede reenvio.
  • Gmail não casa → confira em /filtros se o padrão está exatamente como aparece. Lembre: caractere – (en-dash) é diferente de - (hífen).
  • Fonte URL retorna nada → muitos sites bloqueiam scraping. Tente passar um seletor CSS ou usar o RSS oficial em vez da página.
  • Login não aceita → use o pra conferir o que digitou; desligue autocompletar do navegador.
